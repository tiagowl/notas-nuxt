# Pesquisa de Usuário - Sistema de Gerenciamento de Notas

## 1. Identificação de Personas

### Persona 1: Estudante Universitário

#### Perfil Demográfico
- **Idade**: 18-25 anos
- **Localização**: Principalmente urbana
- **Contexto**: Estudantes de graduação e pós-graduação
- **Dispositivos**: Smartphone (principal), laptop (secundário)

#### Perfil Psicográfico
- **Objetivos**: Organizar estudos, criar resumos, preparar trabalhos acadêmicos
- **Motivações**: Otimizar tempo de estudo, melhorar desempenho acadêmico
- **Frustrações**: Dificuldade em encontrar informações rapidamente, falta de organização

#### Necessidades Identificadas
1. Organizar conteúdo por disciplinas/matérias
2. Gerar resumos automáticos de textos longos
3. Acesso rápido em dispositivos móveis
4. Busca eficiente de notas antigas

#### Jornada do Usuário Típica
1. **Consciência**: Precisa organizar materiais de estudo
2. **Consideração**: Busca ferramentas de organização de notas
3. **Decisão**: Começa a usar o sistema
4. **Uso**: Cria marcadores por disciplina, sub-marcadores por tópicos
5. **Retenção**: Usa IA para gerar resumos de textos longos
6. **Recomendação**: Compartilha com colegas de turma

---

### Persona 2: Profissional de Conhecimento

#### Perfil Demográfico
- **Idade**: 25-45 anos
- **Localização**: Principalmente corporativa/escritórios
- **Contexto**: Gestores de conhecimento, pesquisadores, consultores
- **Dispositivos**: Desktop (principal), tablet (secundário)

#### Perfil Psicográfico
- **Objetivos**: Organizar informações complexas, criar base de conhecimento
- **Motivações**: Eficiência profissional, compartilhamento de conhecimento
- **Frustrações**: Informações fragmentadas, dificuldade em estruturar conhecimento

#### Necessidades Identificadas
1. Estrutura hierárquica clara e escalável
2. Busca eficiente por palavras-chave
3. Edição rápida e intuitiva
4. Organização por projetos/áreas de conhecimento

#### Jornada do Usuário Típica
1. **Consciência**: Necessidade de organizar conhecimento disperso
2. **Consideração**: Avalia ferramentas de gestão de conhecimento
3. **Decisão**: Adota o sistema para projeto específico
4. **Uso**: Cria estrutura hierárquica profunda (marcadores → sub-marcadores → notas)
5. **Retenção**: Usa IA para síntese de documentos extensos
6. **Recomendação**: Implementa em equipe/departamento

---

### Persona 3: Criador de Conteúdo

#### Perfil Demográfico
- **Idade**: 20-40 anos
- **Localização**: Variada (home office, cafés, espaços de co-working)
- **Contexto**: Bloggers, escritores, criadores digitais, jornalistas
- **Dispositivos**: Laptop (principal), smartphone (mobilidade)

#### Perfil Psicográfico
- **Objetivos**: Criar conteúdo de qualidade, organizar ideias, gerenciar projetos editoriais
- **Motivações**: Produtividade criativa, organização de ideias
- **Frustrações**: Bloqueio criativo, dificuldade em estruturar ideias

#### Necessidades Identificadas
1. Editor rich text completo para formatação
2. Geração de conteúdo com IA como apoio criativo
3. Organização por temas/projetos
4. Acesso multiplataforma (mobile e desktop)

#### Jornada do Usuário Típica
1. **Consciência**: Precisa de ferramenta para organizar ideias e projetos
2. **Consideração**: Busca ferramentas com IA integrada
3. **Decisão**: Testa geração de conteúdo com IA
4. **Uso**: Cria marcadores por tema, usa IA para esboços iniciais
5. **Retenção**: Integra no fluxo de trabalho criativo diário
6. **Recomendação**: Menciona em conteúdo sobre produtividade

## 2. Mapeamento de Jornada do Usuário

### Fase 1: Descoberta e Primeiro Contato

#### Touchpoints
- Landing page / página inicial
- Tutorial/onboarding (primeira vez)
- Primeira criação de marcador

#### Emoções
- **Expectativa**: Curiosidade sobre funcionalidades
- **Ansiedade**: Medo de complexidade
- **Confiança**: Interface limpa e intuitiva

#### Pontos de Dor Potenciais
- ❌ Não entender a hierarquia (Marcador → Sub-marcador → Nota)
- ❌ Falta de feedback visual durante ações
- ❌ Onboarding muito longo ou confuso

#### Oportunidades de Melhoria
- ✅ Tutorial interativo guiado
- ✅ Exemplos pré-populados
- ✅ Dicas contextuais

---

### Fase 2: Aprendizado e Exploração

#### Touchpoints
- Criação do primeiro sub-marcador
- Primeira nota criada manualmente
- Primeiro uso da geração com IA

#### Emoções
- **Empolgação**: Descobrindo funcionalidades
- **Confusão**: Entendendo a hierarquia
- **Satisfação**: IA gerando conteúdo útil

#### Pontos de Dor Potenciais
- ❌ Não saber onde criar sub-marcador (dentro de qual marcador)
- ❌ IA demorando muito (>30s) sem feedback
- ❌ Editor rich text complexo para iniciantes

#### Oportunidades de Melhoria
- ✅ Indicadores visuais claros da hierarquia
- ✅ Feedback de progresso durante geração IA
- ✅ Editor rich text com barra de ferramentas intuitiva

---

### Fase 3: Uso Regular

#### Touchpoints
- Acesso diário ao sistema
- Criação/edição de múltiplas notas
- Busca por notas antigas
- Uso em dispositivos móveis

#### Emoções
- **Produtividade**: Fluxo de trabalho estabelecido
- **Frustração**: Dificuldade em encontrar notas antigas
- **Satisfação**: Sistema funcionando bem

#### Pontos de Dor Potenciais
- ❌ Muitas notas sem busca eficiente
- ❌ Interface mobile não otimizada
- ❌ Falta de atalhos de teclado

#### Oportunidades de Melhoria
- ✅ Funcionalidade de busca (futuro)
- ✅ Layout mobile otimizado
- ✅ Atalhos de teclado para ações comuns

---

### Fase 4: Uso Avançado

#### Touchpoints
- Organização de grande volume de notas
- Uso intensivo de IA
- Integração no fluxo de trabalho

#### Emoções
- **Confiabilidade**: Sistema essencial para trabalho
- **Dependência**: Funcionalidade de IA crítica
- **Loyalty**: Sistema integrado ao dia a dia

#### Pontos de Dor Potenciais
- ❌ Limites de taxa da API IA
- ❌ Performance em grandes volumes
- ❌ Falta de exportação de dados

#### Oportunidades de Melhoria
- ✅ Feedback sobre limites de API
- ✅ Otimizações de performance
- ✅ Funcionalidade de exportação (futuro)

## 3. Identificação de Dores e Necessidades

### Dores Principais

#### Dor 1: Complexidade da Hierarquia
**Descrição**: Usuários podem não entender imediatamente a estrutura de 3 níveis  
**Impacto**: Alto - pode causar abandono no primeiro uso  
**Mitigação**: 
- Onboarding visual explicativo
- Indicadores de breadcrumb claros
- Feedback visual ao criar cada nível

#### Dor 2: Geração de IA Lenta
**Descrição**: Tempo de espera durante geração de conteúdo (>10s)  
**Impacto**: Médio - pode frustrar usuários impacientes  
**Mitigação**:
- Feedback de progresso (spinner, barra)
- Estimativa de tempo
- Permitir cancelamento

#### Dor 3: Editor Rich Text Complexo
**Descrição**: Curva de aprendizado do editor  
**Impacto**: Médio - pode intimidar usuários não técnicos  
**Mitigação**:
- Interface intuitiva do shadcn ui vue
- Barra de ferramentas clara
- Dicas contextuais

#### Dor 4: Navegação em Mobile
**Descrição**: Hierarquia pode ser difícil de navegar em telas pequenas  
**Impacto**: Alto - perda de usuários mobile  
**Mitigação**:
- Menu hamburger bem estruturado
- Breadcrumbs móveis
- Gestos de navegação intuitivos

---

### Necessidades Principais

#### Necessidade 1: Organização Visual Clara
**Prioridade**: Alta  
**Solução**: 
- Árvore de navegação hierárquica visível
- Indicadores visuais de nível (cores, ícones)
- Breadcrumbs sempre visíveis

#### Necessidade 2: Feedback Imediato
**Prioridade**: Alta  
**Solução**:
- Loading states em todas as ações
- Mensagens de sucesso/erro claras
- Estados visuais (hover, active, disabled)

#### Necessidade 3: Acesso Rápido
**Prioridade**: Média  
**Solução**:
- Navegação eficiente entre níveis
- Atalhos de teclado (futuro)
- Histórico de notas recentes (futuro)

#### Necessidade 4: Personalização
**Prioridade**: Baixa (futuro)  
**Solução**:
- Temas personalizáveis
- Preferências de ordenação
- Layouts alternativos

## 4. Validação de Hipóteses

### Hipótese 1: Usuários entenderão a hierarquia rapidamente
**Status**: ⚠️ Precisa validação  
**Métrica**: Taxa de criação bem-sucedida de primeiro sub-marcador  
**Critério de Sucesso**: >80% dos usuários criam sub-marcador no primeiro uso  
**Teste**: Onboarding A/B testing

### Hipótese 2: IA será diferencial competitivo
**Status**: ✅ Validadas por requisitos  
**Métrica**: Taxa de uso de geração IA  
**Critério de Sucesso**: >50% dos usuários usam IA pelo menos uma vez  
**Teste**: Analytics de uso

### Hipótese 3: Mobile será importante
**Status**: ✅ Validada por requisitos  
**Métrica**: % de uso em dispositivos móveis  
**Critério de Sucesso**: >40% de uso mobile  
**Teste**: Analytics de dispositivos

### Hipótese 4: Editor rich text é necessário
**Status**: ✅ Validada por requisitos  
**Métrica**: Uso de formatação vs texto simples  
**Critério de Sucesso**: >60% usam formatação  
**Teste**: Analytics de edição

## 5. Oportunidades de Melhoria

### Melhorias Imediatas (MVP)
1. **Onboarding Interativo**: Tutorial guiado passo a passo
2. **Feedback Visual Robusto**: Loading states, mensagens claras
3. **Hierarquia Visual**: Árvore de navegação clara
4. **Breadcrumbs**: Sempre visíveis para contexto

### Melhorias Futuras (Pós-MVP)
1. **Busca Global**: Pesquisa por conteúdo de notas
2. **Tags/Categorias**: Sistema de tags além da hierarquia
3. **Atalhos de Teclado**: Navegação rápida
4. **Exportação**: PDF, Markdown, etc.
5. **Colaboração**: Compartilhamento de notas (futuro)
6. **Temas**: Modo claro/escuro personalizável

## 6. Métricas de Sucesso da Pesquisa

### Métricas Quantitativas
- Taxa de conclusão de onboarding: >85%
- Taxa de criação de primeira nota: >90%
- Taxa de uso de IA: >50%
- Tempo médio para criar primeira nota: <5 minutos
- Taxa de retenção (D7): >40%
- Taxa de retenção (D30): >25%

### Métricas Qualitativas
- NPS (Net Promoter Score): >50
- Satisfação geral: >4.0/5.0
- Facilidade de uso: >4.0/5.0
- Recomendação: >70% recomendariam

### Métricas de Acessibilidade
- Compatibilidade com leitores de tela: 100%
- Contraste de cores (WCAG AA): 100%
- Navegação por teclado: 100%
- Suporte a zoom até 200%: 100%

## 7. Recomendações de Design

### Prioridade 1: Clareza e Simplicidade
- Interface limpa seguindo design system shadcn ui vue
- Hierarquia visual clara (tipografia, espaçamento)
- Cores para diferenciação de níveis (mas não exagerado)

### Prioridade 2: Feedback e Estados
- Loading states em todas as ações assíncronas
- Mensagens de erro amigáveis e acionáveis
- Confirmações para ações destrutivas

### Prioridade 3: Responsividade
- Mobile-first (com melhorias para desktop)
- Touch targets mínimos de 44x44px
- Menu adaptativo (hamburger em mobile)

### Prioridade 4: Acessibilidade
- Contraste adequado (WCAG AA)
- Navegação por teclado completa
- Labels descritivos para leitores de tela
- Foco visível em elementos interativos
